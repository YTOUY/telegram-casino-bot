# Обновление файлов мини-аппа на сервере

## ⚠️ ВАЖНО

**Файлы мини-аппа на сервере НЕ используются для работы мини-аппа!**

Мини-апп работает через Netlify (`https://arbuzcas.netlify.app`), а не с сервера.

Файлы на сервере нужны только для:
- Резервной копии
- Локального тестирования (если настроен веб-сервер)
- Хранения вместе с кодом бота

## Если все равно хотите обновить файлы на сервере

### Вариант 1: Обновить только app.js

**На вашем компьютере (PowerShell):**
```powershell
scp mini_app/app.js root@141.8.198.144:/opt/telegram_bot_test/mini_app/app.js
```

### Вариант 2: Обновить всю папку mini_app

**На вашем компьютере (PowerShell):**
```powershell
scp -r mini_app/* root@141.8.198.144:/opt/telegram_bot_test/mini_app/
```

### Вариант 3: Загрузить все файлы заново

**На вашем компьютере (PowerShell, из папки `тг каз для 11`):**
```powershell
scp -r * root@141.8.198.144:/opt/telegram_bot_test/
```

⚠️ Это перезапишет ВСЕ файлы на сервере!

## Проверка после загрузки

**На сервере (SSH):**
```bash
cd /opt/telegram_bot_test/mini_app
ls -la app.js
# Проверьте дату изменения файла
```

## Что НЕ нужно делать

❌ НЕ нужно настраивать веб-сервер (nginx) для мини-аппа, если используете Netlify
❌ НЕ нужно менять URL в боте - он должен указывать на Netlify
❌ НЕ нужно перезапускать бота после обновления файлов мини-аппа на сервере

## Текущая конфигурация

- **Мини-апп:** https://arbuzcas.netlify.app (Netlify)
- **API сервер:** http://141.8.198.144:8081 (тестовый бот)
- **Netlify Function:** Проксирует запросы от Netlify к API серверу

Файлы на сервере в `/opt/telegram_bot_test/mini_app/` - это просто резервная копия.
